- 统一错误响应体结构为 `{ error: { code, message } }`，业务码如 VALIDATION_ERROR、INVALID_CREDENTIALS、UNAUTHORIZED、FORBIDDEN、SERVER_ERROR 等以大写常量形式返回。
- 受保护路由通过在 handler 前串联 `authMiddleware` 实现鉴权，未登录或 token 失效一律返回 401 并附带中文提示。
- 分页采用 `page/pageSize` 查询参数，统一做 `Math.max(1, ...)` 下界与 `Math.min(50, ...)` 上界保护，再计算 `offset = (page - 1) * pageSize` 配合 LIMIT/OFFSET。
- 所有 SQL 使用 `$1/$2` 占位符并通过数组传参，禁止字符串拼接，防止 SQL 注入。
- 每个路由 handler 包裹 try/catch，异常时记录 `console.error` 并返回 500 + `SERVER_ERROR` 响应。
