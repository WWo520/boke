- 所有写操作路由在 handler 前插入 `authMiddleware`，并在业务逻辑中用 `req.user.id` 做资源归属校验（如 `post.authorId !== req.user.id` 返回 403）。
- 错误响应统一包装为 `{ error: { code, message } }` 或 `{ error: { message } }` 结构，常见 code 包括 `VALIDATION_ERROR`、`NOT_FOUND`、`FORBIDDEN`、`SERVER_ERROR`、`ALREADY_PUBLISHED`。
- 分页参数从 `req.query.page/pageSize` 解析，强制限制 `pageSize ∈ [1,50]`，并通过 `LIMIT $N OFFSET $M` 配合 `COUNT(*)` 计算 `totalPages`。
- SQL 全部使用 `$1/$2` 占位符拼接参数数组，禁止字符串插值直接拼入查询语句。
- 每个路由 handler 包裹 try/catch，捕获后记录 `console.error` 并以 500 + 统一错误体返回。
- 对外 JSON 数据体统一以 `data` 字段承载成功结果，分页额外附带 `pagination: { page, pageSize, total, totalPages }`。
