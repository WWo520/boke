- 所有 SQL 均使用 `$1/$2...` 参数化占位符，禁止字符串拼接用户输入，避免 SQL 注入。
- 对外返回的行对象一律经过 `camelizeRow` 转换，使上层代码始终面对 camelCase 字段（如 `postId`, `createdAt`）。
- 错误处理采用 try/catch 包裹业务逻辑，统一返回 `{ error: { code, message } }` 结构并以 `console.error` 记录堆栈。
- 管理员接口通过链式挂载 `authMiddleware` 后再挂 `adminMiddleware` 实现角色校验，普通鉴权接口仅挂载 `authMiddleware`。
- DDL 使用 `CREATE TABLE IF NOT EXISTS` 配合多次 `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` 实现幂等迁移，重复启动不会报错。
