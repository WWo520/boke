采用 Next.js App Router 作为路由与渲染层：`src/app/` 下按目录映射 URL 路径（如 `post/[slug]/page.jsx`、`u/[username]/write/page.jsx`），根布局 `layout.jsx` 注入全局元信息、字体与 `<Providers>` 包裹；`providers.jsx` 自上而下嵌套 ThemeProvider → AuthProvider → ToastProvider，构成全局状态入口。

- API 层集中在 `api/client.js`，通过统一 `request` 封装 fetch，自动从 `sessionStorage` 读取 `blog_token` 并注入 Authorization 头，再按领域导出 `auth`、`postsApi`、`commentsApi`、`columnsApi`、`questionsApi`、`adminApi` 等命名空间对象，所有请求走 `/api` 代理到后端。
- 全局认证状态由 `context/AuthContext.tsx` 管理，使用 `sessionStorage` 持久化 `blog_user` 与 `blog_token`，并通过 `useAuth()` hook 暴露给子树。
- 可复用 UI 以 `components/<Name>/` 自包含结构组织，每个文件夹内放 `.jsx` 组件与同名 `.module.css` 样式，遵循 CSS Modules 隔离。
- `css_pages/` 为遗留的 react-router-dom 风格页面集合（仍用 `useParams`/`useNavigate`），与 App Router 并存但不再新增；新页面优先放在 `app/` 下。
- SEO 相关静态文件 `sitemap.js`、`robots.js` 置于 `app/` 根目录供 Next.js 自动发现。
- 依赖方向单向：`app/*` → `components/*` / `context/*` / `api/client.js` / `utils/helpers.js`，`api/client.js` 不反向引用任何 UI 代码。
