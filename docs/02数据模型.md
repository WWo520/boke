## 二、表结构详解

### 2.1 users（用户表）

**用途**：存储用户账户信息

```sql
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  name TEXT NOT NULL,
  email TEXT UNIQUE NOT NULL,
  password TEXT NOT NULL,
  avatar TEXT,
  bio TEXT DEFAULT '',
  role TEXT DEFAULT 'user',
  "createdAt" TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**字段说明**：

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PRIMARY KEY | 用户唯一标识 |
| name | TEXT | NOT NULL | 用户名（显示名） |
| email | TEXT | UNIQUE, NOT NULL | 邮箱（登录用） |
| password | TEXT | NOT NULL | bcrypt 哈希密码 |
| avatar | TEXT | - | 头像 URL |
| bio | TEXT | DEFAULT '' | 用户简介 |
| role | TEXT | DEFAULT 'user' | 用户角色（user/admin） |
| createdAt | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**索引**：
- `users_email_idx` (email) - 登录查询优化

**角色说明**：
- `user`：普通用户，可浏览、评论、发布文章
- `admin`：管理员，可管理所有内容

### 2.2 posts（文章表）

**用途**：存储文章内容

```sql
CREATE TABLE posts (
  id SERIAL PRIMARY KEY,
  slug TEXT UNIQUE NOT NULL,
  title TEXT NOT NULL,
  summary TEXT DEFAULT '',
  content TEXT DEFAULT '',
  "coverImage" TEXT DEFAULT '',
  "categoryId" INTEGER REFERENCES categories(id),
  "authorId" INTEGER REFERENCES users(id),
  status TEXT DEFAULT 'published',
  views INTEGER DEFAULT 0,
  "publishedAt" TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  "updatedAt" TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**字段说明**：

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PRIMARY KEY | 文章唯一标识 |
| slug | TEXT | UNIQUE, NOT NULL | 文章 URL 标识（标题转小写 + 时间戳） |
| title | TEXT | NOT NULL | 文章标题 |
| summary | TEXT | DEFAULT '' | 文章摘要 |
| content | TEXT | DEFAULT '' | 文章正文（Markdown） |
| coverImage | TEXT | DEFAULT '' | 封面图 URL |
| categoryId | INTEGER | REFERENCES categories(id) | 分类 ID |
| authorId | INTEGER | REFERENCES users(id) | 作者 ID |
| status | TEXT | DEFAULT 'published' | 状态（published/draft） |
| views | INTEGER | DEFAULT 0 | 阅读次数 |
| publishedAt | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 发布时间 |
| updatedAt | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

**索引**：
- `idx_posts_slug` (slug) - 文章详情查询优化
- `idx_posts_categoryId` (categoryId) - 分类查询优化
- `idx_posts_authorId` (authorId) - 用户文章查询优化

**状态说明**：
- `published`：已发布，公开可见
- `draft`：草稿，仅作者可见

### 2.29 视图定义

**post_rankings（文章排行榜视图）**：
```sql
CREATE VIEW post_rankings AS
SELECT 
  p.id,
  p.title,
  p.slug,
  p."coverImage",
  p.views,
  p."likeCount",
  p."favoriteCount",
  (SELECT COUNT(*) FROM comments WHERE "postId" = p.id) as commentCount,
  p.hotScore,
  p."authorId",
  u.name as authorName,
  u.avatar as authorAvatar,
  p."publishedAt"
FROM posts p
JOIN users u ON p."authorId" = u.id
WHERE p.status = 'published' AND p."reviewStatus" = 'approved'
ORDER BY p.hotScore DESC, p.views DESC;
```

**author_rankings（作者排行榜视图）**：
```sql
CREATE VIEW author_rankings AS
SELECT 
  u.id,
  u.name,
  u.avatar,
  u.bio,
  u.level,
  u.points,
  COUNT(DISTINCT p.id) as postCount,
  SUM(p.views) as totalViews,
  SUM(p."likeCount") as totalLikes,
  (SELECT COUNT(*) FROM user_follows WHERE "followId" = u.id) as followersCount
FROM users u
LEFT JOIN posts p ON u.id = p."authorId" AND p.status = 'published' AND p."reviewStatus" = 'approved'
GROUP BY u.id, u.name, u.avatar, u.bio, u.level, u.points
ORDER BY followersCount DESC, totalViews DESC;
```

**tag_rankings（标签排行榜视图）**：
```sql
CREATE VIEW tag_rankings AS
SELECT 
  tag,
  COUNT(*) as postCount,
  SUM(p.views) as totalViews
FROM post_tags pt
JOIN posts p ON pt."postId" = p.id AND p.status = 'published' AND p."reviewStatus" = 'approved'
GROUP BY tag
ORDER BY postCount DESC, totalViews DESC;
```

### 5.2 数据转换

**蛇形命名 → 驼峰命名**：

```javascript
function snakeToCamel(str) {
  return str.replace(/(_\w)/g, (m) => m[1].toUpperCase());
}

function camelizeRow(row) {
  const result = {};
  for (const key of Object.keys(row)) {
    result[snakeToCamel(key)] = row[key];
  }
  return result;
}
```
