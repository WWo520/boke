### 1.2 架构特点

| 特点 | 描述 |
|------|------|
| **三层架构** | 客户端层 → API 网关层 → 数据访问层 |
| **无状态认证** | JWT Bearer 认证，无需服务端会话 |
| **模块化设计** | 前端路由按需加载，后端路由按模块划分 |
| **数据库连接池** | PostgreSQL 连接复用，提升性能 |
| **统一错误处理** | 前端 API 客户端统一错误格式 |

## 二、分层架构说明

### 2.1 客户端层

**职责**：处理用户交互、页面渲染、状态管理

**技术栈**：React 18 + React Router 6 + CSS Modules

**核心能力**：
- 页面路由管理（懒加载）
- 用户认证状态管理
- API 请求封装
- 响应式 UI 渲染
- SEO Meta 标签管理

### 2.2 API 网关层

**职责**：接收请求、路由分发、认证授权、响应处理

**技术栈**：Express 4 + JWT + bcryptjs

**核心能力**：
- RESTful API 路由定义
- JWT 令牌验证
- 用户角色权限控制
- 文件上传处理
- 错误响应统一格式

### 2.3 数据访问层

**职责**：数据库连接、SQL 执行、数据查询

**技术栈**：PostgreSQL + pg

**核心能力**：
- 数据库连接池管理
- 参数化 SQL 查询
- 数据插入/更新/删除
- 索引优化
- 事务支持

## 三、数据流设计

### 3.1 认证数据流

```
用户登录 → POST /api/auth/login → bcrypt.verify() → JWT.sign() → 返回 token → localStorage 存储
```

**详细流程**：
1. 用户输入邮箱密码
2. 前端调用 `auth.login(email, password)`
3. 后端验证邮箱是否存在
4. 使用 bcrypt 验证密码
5. 生成 JWT 令牌（7天有效期）
6. 返回 token 和用户信息
7. 前端存储到 localStorage

### 3.2 文章发布数据流

```
写文章页面 → 表单验证 → 图片上传 → POST /api/posts → 数据库插入 → 返回文章信息
```

**详细流程**：
1. 用户填写文章信息（标题、摘要、正文、标签）
2. 上传封面图（前端 Canvas 压缩）
3. 选择状态（发布/草稿）
4. 前端调用 `postsApi.create(data)`
5. 后端验证参数
6. 生成 slug（标题转小写 + 时间戳）
7. 插入 posts 表
8. 插入 post_tags 关联表
9. 返回文章数据

### 3.3 文章浏览数据流

```
首页请求 → GET /api/posts → 查询 posts + categories + users → 返回分页数据 → 前端渲染
```

**详细流程**：
1. 前端发起 GET `/api/posts?page=1&pageSize=9`
2. 后端构建 SQL 查询（JOIN categories、LEFT JOIN users）
3. 添加状态过滤（status = 'published'）
4. 查询标签和评论数
5. 返回分页数据
6. 前端渲染文章列表

## 四、关键设计决策

### 4.1 认证设计

- **JWT 无状态认证**：服务端无需存储会话，便于水平扩展
- **Token 存储**：localStorage 存储，支持跨标签页共享
- **Token 有效期**：7天，过期后需重新登录
- **密码安全**：bcryptjs 哈希（10轮），防止彩虹表攻击

### 4.2 路由设计

- **用户个人页面**：统一使用 `/u/:username/` 前缀，便于 SEO 和用户识别
- **管理页面**：`/admin` 独立路由，需管理员权限
- **懒加载**：React Router 动态 import，减少首屏加载体积

### 4.3 数据库设计

- **关系型数据库**：PostgreSQL 保证数据一致性
- **参数化查询**：防止 SQL 注入
- **连接池**：复用数据库连接，减少连接开销
- **索引优化**：posts(slug)、posts(categoryId)、posts(authorId)、post_tags(postId)、comments(postId)

### 4.4 文件上传设计

- **前端压缩**：Canvas 压缩到 1920px 最长边，质量 0.85
- **文件名随机化**：时间戳 + 随机字符串，防止覆盖
- **存储位置**：`server/uploads/` 目录
- **访问路径**：`/uploads/filename`

## 五、部署架构

### 5.1 开发环境

| 服务 | 端口 | 说明 |
|------|------|------|
| 前端开发服务器 | 3000 | Vite dev server |
