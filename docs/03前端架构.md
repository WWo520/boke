## 三、路由设计

### 3.1 路由配置（Next.js App Router 文件系统路由）

| 路由 | 权限 | 说明 |
|------|------|------|
| `/` | 公开 | 首页，展示文章列表 |
| `/page/:pageNum` | 公开 | 分页文章列表 |
| `/category/:slug` | 公开 | 分类文章列表 |
| `/category/:slug/page/:pageNum` | 公开 | 分类分页 |
| `/post/:slug` | 公开 | 文章详情 |
| `/about` | 公开 | 关于页面 |
| `/admin` | 管理员 | **管理面板**（Tailwind CSS 样式） |
| `/u/:username` | 公开 | 用户个人页面 |
| `/u/:username/posts` | 公开 | 用户文章列表 |
| `/u/:username/profile` | 登录用户 | 个人资料管理 |
| `/u/:username/write` | 登录用户 | **创作页面**（写文章） |
| `/u/:username/write/:id` | 登录用户 | **创作页面**（编辑文章） |
| `/questions` | 公开 | 问答页面 |
| `/rankings` | 公开 | 排行榜 |
| `*` | 公开 | 404页面 |

### 3.2 路由守卫

**认证守卫**（`useAuth` Hook / `AuthContext`）：
- 检查 localStorage 中是否存在 token
- 验证 token 是否过期
- 在组件中使用 `useAuth()` 获取当前用户
- 未登录时重定向到登录页面

**管理员守卫**（`adminMiddleware`）：
- 后端 `authMiddleware` + `adminMiddleware` 双重保护
- 前端通过 `isAdmin` 判断角色权限

## 四、样式方案

项目使用两种样式方案共存：

### 4.1 CSS Modules（主要方案）

大部分页面使用 CSS Modules 实现，样式文件位于 `src/css_pages/*.module.css`。

```jsx
import styles from './Admin.module.css';
<div className={styles.page}>...</div>
```

### 4.2 Tailwind CSS（管理面板）

管理面板 (`/admin`) 使用 **Tailwind CSS v4** 原子化样式类。

**配置入口**：
- [globals.css](file:///d:/develop/trae/blog_demo/src/app/globals.css) — 顶部导入 `@import "tailwindcss"`
- [postcss.config.mjs](file:///d:/develop/trae/blog_demo/postcss.config.mjs) — PostCSS 插件配置

```css
/* globals.css */
@import "tailwindcss";
```

```js
// postcss.config.mjs
const config = {
  plugins: {
    '@tailwindcss/postcss': {},
  },
};
export default config;
```

**使用方式**：
```tsx
// src/app/admin/page.tsx
<div className="min-h-screen bg-gray-50">
  <div className="bg-gradient-to-r from-blue-600 via-indigo-600 to-purple-600 py-12">
    ...
  </div>
</div>
```

**注意**：Tailwind CSS v4 无需独立的 `tailwind.config.*` 文件，PostCSS 插件会自动扫描 `src` 目录下的类名并生成对应的样式。

## 六、API 客户端设计

### 6.1 请求封装（`api/client.js`）

```javascript
const BASE_URL = '/api';

async function request(url, options = {}) {
  const token = localStorage.getItem('token');
  const headers = {
    'Content-Type': 'application/json',
    ...options.headers,
  };
  if (token) headers['Authorization'] = `Bearer ${token}`;
  const response = await fetch(`${BASE_URL}${url}`, { ...options, headers });
  const data = await response.json();
  if (!response.ok) throw new Error(data.error?.message || '请求失败');
  return data;
}
```

## 十、已知问题与注意事项

### 10.1 导航返回

创作页面（`/u/:username/write` 和 `/u/:username/write/:id`）以及个人资料页面的返回按钮，应使用 `router.back()` 而不是 `router.push(-1)`，因为 `router.push()` 只接受 URL 字符串参数。

✅ 正确用法：
```jsx
<button onClick={() => router.back()}>返回</button>
```

❌ 错误用法：
```jsx
<button onClick={() => router.push(-1)}>返回</button>  // 不会触发导航
```
